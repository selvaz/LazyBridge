# compare_models Tool Reference

## What It Does

`compare_models` takes a list of run IDs from previous `fit_model` calls and compares them by AIC, BIC, and log-likelihood. It identifies the best model by each criterion and returns a structured comparison. This is the primary tool for model selection when you have fitted multiple competing specifications.

## Tool Signature

```python
compare_models(
    run_ids: list[str],   # Required. List of run ID strings.
)
```

## Parameters

| Parameter | Type | Required | Default | Description |
|---|---|---|---|---|
| `run_ids` | `list[str]` | Yes | -- | A list of 2 or more run ID strings from previous `fit_model` calls. Each run must exist in the MetaStore. Runs that are `None` (not found) are silently filtered out. |

## Prerequisites

Before calling `compare_models`, you need 2 or more successful `fit_model` runs:

1. Fit model A: `result_a = fit_model(family="garch", target_col="ret", dataset_name="equities", params={"p": 1, "q": 1})`
2. Fit model B: `result_b = fit_model(family="garch", target_col="ret", dataset_name="equities", params={"p": 1, "q": 1, "vol": "TARCH"})`
3. Compare: `compare_models(run_ids=[result_a["run_id"], result_b["run_id"]])`

The runs do NOT need to be the same model family. You can compare ARIMA vs GARCH vs OLS, though the comparison is most meaningful when models are fitted on the same data.

Runs with `status: "failed"` can be included -- they will have `None` for AIC/BIC/log-likelihood and will not be selected as "best" by any criterion.

## Return Value: DiagnosticResult

On success, returns a `DiagnosticResult` dict:

```json
{
  "test_name": "Model Comparison",
  "statistic": null,
  "p_value": null,
  "passed": null,
  "detail": {
    "models": [
      {
        "run_id": "a1b2c3d4e5f6g7h8",
        "engine": "garch",
        "dataset": "equities",
        "aic": 1234.56,
        "bic": 1267.89,
        "log_likelihood": -612.28
      },
      {
        "run_id": "x9y8z7w6v5u4t3s2",
        "engine": "garch",
        "dataset": "equities",
        "aic": 1240.12,
        "bic": 1278.45,
        "log_likelihood": -614.06
      }
    ]
  },
  "interpretation": "Compared 2 model runs. Best AIC: garch (run a1b2c3d4, AIC=1234.56). Best BIC: garch (run a1b2c3d4, BIC=1267.89)."
}
```

### Field Descriptions

| Field | Type | Description |
|---|---|---|
| `test_name` | `str` | Always `"Model Comparison"` |
| `statistic` | `null` | Not used for model comparison |
| `p_value` | `null` | Not used for model comparison |
| `passed` | `null` | Not used for model comparison |
| `detail` | `dict` | Contains the `models` list |
| `detail.models` | `list[dict]` | One entry per run. Each has `run_id`, `engine`, `dataset`, `aic`, `bic`, `log_likelihood`. |
| `interpretation` | `str` | Natural-language summary identifying the best model by AIC and BIC |

### Per-Model Entry in detail.models

| Key | Type | Description |
|---|---|---|
| `run_id` | `str` | The full 16-char run ID |
| `engine` | `str` | Model family: `"ols"`, `"arima"`, `"garch"`, `"markov"` |
| `dataset` | `str` | The dataset name used for fitting |
| `aic` | `float \| null` | Akaike Information Criterion. `null` if the metric was not produced (e.g. failed run). |
| `bic` | `float \| null` | Bayesian Information Criterion. `null` if not available. |
| `log_likelihood` | `float \| null` | Log-likelihood of the fitted model. `null` if not available. |

### The interpretation Field

The interpretation string follows this pattern:

```
"Compared N model runs. Best AIC: {engine} (run {run_id[:8]}, AIC={value:.2f}). Best BIC: {engine} (run {run_id[:8]}, BIC={value:.2f})."
```

If no models have valid AIC or BIC (e.g. all runs failed), the interpretation is just `"Compared N model runs."` without best-model sentences.

### Error Returns

```json
{"error": true, "message": "No valid runs found"}
```

This is returned when all provided run IDs resolve to `None` (not found in the MetaStore).

```json
{"error": true, "type": "ExceptionType", "message": "..."}
```

Returned for unexpected errors.

## Auto-Generated Plot

The `compare_models` tool itself does NOT generate a comparison plot. However, the `plots.plot_model_comparison()` function exists in the runtime and can be called programmatically. When using the tool layer, no plot is auto-generated by this tool.

If a model comparison plot was generated through the runner (e.g. as part of a workflow), it would be named `model_comparison` and stored as a plot artifact. The chart is a grouped bar chart with:
- X-axis: model labels formatted as `"{engine}\n({run_id[:6]})"`
- Blue bars: AIC values
- Orange bars: BIC values
- Value labels above each bar
- Width scales with the number of models

## Setting Up a Comparison Workflow

### Step-by-Step

```python
# Step 1: Register data
register_dataset(name="equities", uri="/data/returns.parquet", time_column="date")

# Step 2: Fit multiple models
run_a = fit_model(family="garch", target_col="ret", dataset_name="equities",
                  params={"p": 1, "q": 1})

run_b = fit_model(family="garch", target_col="ret", dataset_name="equities",
                  params={"p": 1, "q": 1, "vol": "TARCH"})

run_c = fit_model(family="garch", target_col="ret", dataset_name="equities",
                  params={"p": 2, "q": 1, "dist": "t"})

# Step 3: Collect run IDs (only from successful runs)
run_ids = []
for r in [run_a, run_b, run_c]:
    if r.get("status") == "success":
        run_ids.append(r["run_id"])

# Step 4: Compare
comparison = compare_models(run_ids=run_ids)

# Step 5: Read results
best_aic_model = None
for m in comparison["detail"]["models"]:
    if m["aic"] is not None:
        if best_aic_model is None or m["aic"] < best_aic_model["aic"]:
            best_aic_model = m

print(f"Best model: {best_aic_model['engine']} (run {best_aic_model['run_id']})")
```

### Comparing Across Families

```python
run_arima = fit_model(family="arima", target_col="ret", dataset_name="equities",
                      params={"order": [1, 0, 1]})

run_garch = fit_model(family="garch", target_col="ret", dataset_name="equities",
                      params={"p": 1, "q": 1})

comparison = compare_models(run_ids=[run_arima["run_id"], run_garch["run_id"]])
```

Cross-family comparison is valid when the models are fitted on the same data and the same target variable. AIC and BIC are comparable as long as the log-likelihood is computed on the same observations.

**Caveat**: AIC/BIC are NOT directly comparable when:
- Models are fitted on different data (different datasets or different SQL filters).
- One model applies differencing (e.g. ARIMA with d=1) and the other does not, because the effective sample size differs.
- The models use different error distributions (e.g. normal vs Student-t) -- the log-likelihoods are on different scales.

## Interpreting AIC/BIC Differences

When comparing two models, the absolute AIC/BIC value is less important than the difference.

### AIC Difference Scale (Burnham & Anderson)

| AIC Difference (Delta) | Evidence Against Higher-AIC Model |
|---|---|
| 0 -- 2 | Essentially no difference; both models are plausible |
| 2 -- 4 | Weak evidence favoring the lower-AIC model |
| 4 -- 7 | Moderate evidence favoring the lower-AIC model |
| 7 -- 10 | Strong evidence favoring the lower-AIC model |
| > 10 | Decisive; the higher-AIC model has essentially no support |

### BIC Difference Scale (Kass & Raftery)

| BIC Difference (Delta) | Evidence Against Higher-BIC Model |
|---|---|
| 0 -- 2 | Weak; not worth mentioning |
| 2 -- 6 | Positive evidence favoring lower-BIC model |
| 6 -- 10 | Strong evidence favoring lower-BIC model |
| > 10 | Very strong / decisive evidence |

### General Guidelines

- **AIC** tends to select more complex models (lower penalty for parameters). Better for prediction.
- **BIC** penalizes complexity more heavily. Better for identifying the "true" model. Preferred when sample size is large.
- When AIC and BIC agree on the best model, the result is robust.
- When they disagree, report both. The AIC-best model may forecast better; the BIC-best model may be more parsimonious.
- **Log-likelihood** alone is not a valid comparison metric because it always favors more complex models (more parameters = better fit). AIC and BIC correct for this.

## Examples

### GARCH(1,1) vs GJR-GARCH(1,1)

```python
# Standard GARCH
run_std = fit_model(family="garch", target_col="ret", dataset_name="equities",
                    params={"p": 1, "q": 1})

# GJR-GARCH (asymmetric, captures leverage effect)
run_gjr = fit_model(family="garch", target_col="ret", dataset_name="equities",
                    params={"p": 1, "q": 1, "vol": "TARCH"})

result = compare_models(run_ids=[run_std["run_id"], run_gjr["run_id"]])

# Example interpretation output:
# "Compared 2 model runs. Best AIC: garch (run x9y8z7w6, AIC=1230.12).
#  Best BIC: garch (run a1b2c3d4, BIC=1260.45)."
```

### ARIMA Order Selection

```python
runs = []
for order in [[1,0,0], [1,1,0], [1,1,1], [2,1,1]]:
    r = fit_model(family="arima", target_col="price", dataset_name="stocks",
                  params={"order": order})
    if r.get("status") == "success":
        runs.append(r["run_id"])

result = compare_models(run_ids=runs)
# detail.models will contain AIC/BIC for each ARIMA order
```

### GARCH Distribution Comparison

```python
runs = []
for dist in ["normal", "t", "skewt"]:
    r = fit_model(family="garch", target_col="ret", dataset_name="equities",
                  params={"p": 1, "q": 1, "dist": dist})
    if r.get("status") == "success":
        runs.append(r["run_id"])

result = compare_models(run_ids=runs)
# Identifies which error distribution best fits the data
```

### OLS vs ARIMA for Trend Modeling

```python
run_ols = fit_model(family="ols", target_col="gdp", dataset_name="macro")
run_arima = fit_model(family="arima", target_col="gdp", dataset_name="macro",
                      params={"order": [1, 1, 0]})

result = compare_models(run_ids=[run_ols["run_id"], run_arima["run_id"]])
```

### Handling Failed Runs

```python
run_a = fit_model(family="garch", target_col="ret", dataset_name="equities",
                  params={"p": 1, "q": 1})
run_b = fit_model(family="garch", target_col="ret", dataset_name="equities",
                  params={"p": 5, "q": 5})  # Might fail to converge

# Include both -- failed runs have null metrics and are skipped for "best" selection
result = compare_models(run_ids=[run_a["run_id"], run_b["run_id"]])

# If run_b failed, its entry in detail.models will have aic: null, bic: null
# The interpretation will still identify run_a as best
```

## Edge Cases

- **Single run**: Technically works (returns a comparison with 1 model) but not useful. Use `get_run` instead.
- **All runs failed**: Returns `detail.models` with all-null metrics. Interpretation is `"Compared N model runs."` with no best-model sentence.
- **Run ID not found**: That run is silently filtered out. If ALL run IDs are not found, returns `{"error": true, "message": "No valid runs found"}`.
- **Mixed families**: Valid and supported. AIC/BIC values are included for all families. See caveats above about comparability.
- **Duplicate run IDs**: The same run appears multiple times in the comparison table. No deduplication is performed.
