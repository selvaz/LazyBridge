## 2. Positioning north star — the 5 strengths

Everything we write must surface these five real, code-backed differentiators:

1. **Multi-dimensional recursive composition.** `Plan` is an engine; an
   `Agent(engine=Plan(...))` is a valid `Tool`, so pipelines nest with no special
   syntax, at any depth. Cost/tokens/OTel roll up automatically
   (`Envelope.metadata.nested_*`).

2. **Pre-launch plan validation.** `PlanCompiler.validate()`
   (`lazybridge/engines/plan/_compiler.py`, ~480 lines) raises `PlanCompileError` at
   construction for: duplicate step names, missing tools, forward references, sentinels
   pointing at siblings in the same parallel band, `from_parallel_all` band-start
   violations, type drift between consecutive steps, malformed `routes=`/`routes_by=`,
   bad `after_branches`. All before any LLM call.

3. **"Everything is a tool" is the architectural glue, not a slogan.**
   `Tool.wrap()` accepts callable / Agent / Tool; one contract for functions,
   sub-agents, plans, MCP servers, native tools.

4. **Structured output + verify as quality control.** `output=PydanticModel`
   (schema-validated, re-prompts on failure) + `output_validator=` (post-parse runtime
   check) + `verify=judge_agent` (`lazybridge/_verify.py`: allowlist verdict
   recognition, feedback flows via context never onto the task, up to `max_verify`
   retries). `verify=` is applicable at agent level, `agent.as_tool(verify=...)` level,
   and Plan-step level.

5. **Observability is native.** Session + EventLog (SQLite, batched, back-pressure
   policies) + `OTelExporter` emitting OpenTelemetry GenAI semconv (`gen_ai.*`
   attributes, `nesting_level`, parent-child spans via contextvars). Opt-out, not
   opt-in.

**Narrative order on marketing surfaces:** 1 → 2 → 3 → 4 → 5.

- **Plans fail fast** — "PlanCompileError catches duplicate names, forward references, type drift, and broken sentinels — at construction, before any LLM call."

**Hero code (tab "Pipelines that nest", default):**

```python
from lazybridge import Agent, LLMEngine, Plan, Step, Session, from_step

search    = Agent(engine=LLMEngine("gpt-5.4-mini"),      name="search")
summarise = Agent(engine=LLMEngine("gemini-2.5-pro"),    name="summarise")
writer    = Agent(engine=LLMEngine("claude-sonnet-4-6"), name="write")

research = Agent(
    engine=Plan(Step("search"), Step("summarise")),       # a sub-pipeline
    tools=[search, summarise], name="research",
)
article = Agent(
    engine=Plan(Step("research"),                          # nested — research is one tool
                Step("write", context=from_step("research"))),
    tools=[research, writer], session=Session(),
)
print(article("AI agents in 2026").text())
```

**Tab "Simplest":**

```python
from lazybridge import Agent, LLMEngine

agent = Agent(engine=LLMEngine("claude-sonnet-4-6"))
print(agent("Explain LazyBridge in one sentence.").text())
```

**Tab "With verify + resume":** nested example plus `verify=judge` on the write step
and `Store(db="run.sqlite")` + `checkpoint_key=` on the outer Plan.

**Code caption:** "The simple case stays one line. The architecture grows only when the problem grows — without changing the mental model."
