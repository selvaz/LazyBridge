One task. Three ways to write it.

Tool loop: the model searches the web and summarises results.

~45 lines
Python
import json
import openai

client = openai.OpenAI()

# 1. Define schema manually
tools = [{
    "type": "function",
    "name": "search_web",
    "description": "Search the web for information",
    "parameters": {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "Search query"
            }
        },
        "required": ["query"],
    },
}]

def search_web(query: str) -> str:
    return f"Results for '{query}': [article1, article2, ...]"

input_messages = [{"role": "user",
    "content": "Find top 3 AI papers this week"}]

# 2. Manual tool-call loop
while True:
    response = client.responses.create(
        model="gpt-4o",
        input=input_messages,
        tools=tools,
    )
    fn_calls = [i for i in response.output
                if i.type == "function_call"]

    if fn_calls:
        input_messages.extend(
            [i.model_dump() for i in response.output])
        for fc in fn_calls:
            args = json.loads(fc.arguments)
            result = search_web(**args)
            input_messages.append({
                "type": "function_call_output",
                "call_id": fc.call_id,
                "output": result,
            })
    else:
        text_item = next(i for i in response.output
                         if i.type == "message")
        print(text_item.content[0].text)
        break

What you're dealing with:

  • ❌ Manual JSON schema dict
  • ❌ While loop with output item iteration
  • ❌ Provider-specific message formats
  • ❌ Manual message list accumulation
  • ❌ Content block extraction per provider
  • ❌ Rewrite for every provider switch
~30 lines
Python
from langchain_openai import ChatOpenAI
from langchain.agents import (
    AgentExecutor,
    create_tool_calling_agent,
)
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.tools import tool

@tool
def search_web(query: str) -> str:
    """Search the web for information."""
    return f"Results for '{query}': [article1, ...]"

llm   = ChatOpenAI(model="gpt-4o")
tools = [search_web]

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_tool_calling_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools)

result = agent_executor.invoke({
    "input": "Find top 3 AI papers this week"
})
print(result["output"])

Trade-offs:

  • ⚠️ @tool decorator + schema wiring
  • ⚠️ AgentExecutor boilerplate
  • ⚠️ LCEL prompt template required
  • ⚠️ Import chain: 5+ modules
  • ⚠️ Frequent breaking changes
  • ✓ Broad ecosystem (RAG, vector DBs)
6 lines
Python
from lazybridge import LazyAgent, LazyTool

def search_web(query: str) -> str:
    """Search the web for information."""
    return f"Results for '{query}': [article1, ...]"

result = LazyAgent("openai").loop(
    "Find top 3 AI papers this week and summarise",
    tools=[LazyTool.from_function(search_web)],
)
print(result.content)

What you get:

  • ✅ Schema from type hints automatically
  • ✅ Tool-call loop handled internally
  • ✅ Message accumulation handled
  • ✅ Change "openai" → any provider
  • ✅ Event tracking for free
  • ✅ No format translation needed

Context Chaining

Wire agents together — before they run

LazyContext.from_agent() reads an agent's last output and injects it into another agent's system prompt at execution time. Declare the entire topology upfront — no side effects, no coupling.

Python
from lazybridge import LazyAgent, LazyContext

# 1. Declare agents and wire context — nothing runs yet
researcher = LazyAgent("anthropic", name="researcher")

fact_checker = LazyAgent("openai", name="fact_checker",
    context=LazyContext.from_agent(researcher))

writer = LazyAgent("anthropic", name="writer",
    context=(
        LazyContext.from_agent(researcher,   prefix="[Research]")
        + LazyContext.from_agent(fact_checker, prefix="[Fact-check]")
        + LazyContext.from_text("Write in plain English, max 300 words.")
    ))

# 2. Execute in order — each agent's _last_output feeds the next
researcher.loop("Key climate research findings this month?")
fact_checker.chat("Verify the claims above. Flag unsourced ones.")
result = writer.chat("Write a briefing based on your context.")
print(result.content)

How it works

1
Lazy declaration

Contexts are wired at construction time. No agent has run. No LLM calls, no side effects — just a dependency graph.

2
Pull, not push

researcher doesn't send anything. fact_checker pulls researcher._last_output when it runs. If the agent hasn't run yet, context is "" — no crash.

3
Composable with +

writer sees both agents' outputs plus a static instruction — all joined with \n\n and injected as one system prompt block.

4
All sources

Mix freely: from_agent() + from_store() + from_text() + from_function(). Any combination works.

You can inspect any context without running an agent:
print(writer.context()) → shows exactly what the LLM will receive

With store + function — all sources combined

Python
from datetime import date
from lazybridge import LazyAgent, LazyContext, LazyStore

store = LazyStore()
store.write("style_guide", "Formal tone. No bullet points. Max 400 words.")

analyst = LazyAgent("anthropic", name="analyst")
reporter = LazyAgent("openai", name="reporter",
    context=(
        LazyContext.from_agent(analyst)                          # previous agent's output
        + LazyContext.from_store(store, keys=["style_guide"])    # shared rules
        + LazyContext.from_function(lambda: f"Date: {date.today()}")  # live data
    ))

analyst.chat("Analyse Q3 earnings for the top 5 tech companies.")
result = reporter.chat("Write the investor briefing.")
print(result.content)
Investment Research Platform

Four architectures, one outcome

Market data → risk analysis → investment report. The same workflow in four different patterns — from LLM-orchestrated to fully typed end-to-end.

Python
from pydantic import BaseModel, Field
from lazybridge import LazyAgent, LazySession, NativeTool

TASK = ("Conduct a full investment analysis for Q2 2025. "
        "Focus on AI infrastructure, energy transition, and CRISPR therapeutics.")

class MarketBriefing(BaseModel):
    sectors: list[str]
    summaries: dict[str, str]
    overall_outlook: str

class RiskProfile(BaseModel):
    overall_sentiment: str
    recommended_action: str
    rationale: str

class InvestmentReport(BaseModel):
    title: str
    executive_summary: str
    top_opportunities: list[str]
    conclusion: str

def run_variant_1():
    # ── Layer 1: 3 Google agents research in parallel ──
    market_sess = LazySession(tracking="basic", console=True)
    LazyAgent("google", name="tech_researcher",    session=market_sess,
              native_tools=[NativeTool.WEB_SEARCH], output_schema=MarketBriefing,
              system="Technology analyst. Output ONLY valid JSON.")
    LazyAgent("google", name="energy_researcher",  session=market_sess,
              native_tools=[NativeTool.WEB_SEARCH], output_schema=MarketBriefing,
              system="Energy analyst. Output ONLY valid JSON.")
    LazyAgent("google", name="biotech_researcher", session=market_sess,
              native_tools=[NativeTool.WEB_SEARCH], output_schema=MarketBriefing,
              system="Biotech analyst. Output ONLY valid JSON.")

    market_tool = market_sess.as_tool(
        "market_intelligence",
        "Three Google agents research tech, energy, and biotech in parallel.",
        mode="parallel",
    )

    # ── Layer 2: chain — risk analyst → report writer ──
    analysis_sess = LazySession(tracking="basic", console=True)
    analysis_chain = analysis_sess.as_tool(
        "analysis_chain",
        "Risk analysis then investment report generation.",
        mode="chain",
        participants=[
            LazyAgent("anthropic", name="risk_analyst",  session=analysis_sess,
                      output_schema=RiskProfile,
                      system="Senior risk analyst. Output ONLY valid JSON."),
            LazyAgent("openai",    name="report_writer", session=analysis_sess,
                      output_schema=InvestmentReport,
                      system="CFA report writer. Output ONLY valid JSON."),
        ],
    )

    # ── Layer 3: orchestrator drives both tools ──
    orchestrator = LazyAgent(
        "anthropic",
        system="Investment research coordinator. Use both tools to produce a full report.",
    )
    result = orchestrator.loop(TASK, tools=[market_tool, analysis_chain], max_steps=6)
    print(result.content)

run_variant_1()

Three-Tier Orchestrator

The LLM decides when to call each tool and in what order. Full orchestration intelligence.

orchestrator (Anthropic)
├── market_intelligence_tool
│   ├── tech_researcher (Google)
│   ├── energy_researcher (Google)
│   └── biotech_researcher (Google)
└── analysis_chain_tool
    ├── risk_analyst (Anthropic)
    └── report_writer (OpenAI)

Key pattern

Two LazySession.as_tool() calls expose multi-agent pipelines as single tools. The orchestrator calls them via loop() — it decides order and iteration.

        participants=[market_tool, risk_analyst, report_writer],

Nested Pipeline, No Orchestrator

The entire pipeline runs as a single tool call. No orchestrator LLM — the framework drives execution.

├── market_tool (parallel LazyTool)
│   ├── tech_researcher
│   ├── energy_researcher
│   └── biotech_researcher

└── report_writer (OpenAI)

Key pattern

A LazyTool is a valid participant in as_tool(mode="chain"). The chain calls .run({"task": ...}) on it and passes the output to the next participant.

from pydantic import BaseModel
from lazybridge import LazyAgent, LazySession, LazyTool, NativeTool

TASK = ("Conduct a full investment analysis for Q2 2025. "
        "Focus on AI infrastructure, energy transition, and CRISPR therapeutics.")

class InvestmentReport(BaseModel):
    title: str
    executive_summary: str
    top_opportunities: list[str]
    top_risks: list[str]
    conclusion: str

def fake_database_query(sector: str) -> str:
    """Query the internal investment database for a sector."""
    return f"[DB] {sector}: PE 24.5, YoY +18%, institutional ownership 67%"

def run_variant_3():
    # ── Generalist: broad sweep with custom tool ──
    generalist = LazyAgent(
        "anthropic", name="generalist",
        system="Investment researcher with database access. Gather broad market intelligence.",
        tools=[LazyTool.from_function(fake_database_query)],
    )

    # ── Sector specialists: run in parallel ──
    sector_sess = LazySession(tracking="basic")
    LazyAgent("google", name="tech_specialist",    session=sector_sess,
              native_tools=[NativeTool.WEB_SEARCH],
              system="Technology sector specialist. Focus on AI, semiconductors, cloud.")
    LazyAgent("google", name="energy_specialist",  session=sector_sess,
              native_tools=[NativeTool.WEB_SEARCH],
              system="Energy sector specialist. Focus on renewables, oil, grid.")
    LazyAgent("google", name="biotech_specialist", session=sector_sess,
              native_tools=[NativeTool.WEB_SEARCH],
              system="Biotech sector specialist. Focus on pharma, genomics, medtech.")

    sector_intel_tool = sector_sess.as_tool(
        "sector_intelligence",
        "Parallel sector research: tech, energy, biotech.",
        mode="parallel",
        combiner="concat",
    )

    # ── Synthesizer: combines all inputs into final report ──
    synthesizer = LazyAgent(
        "openai", name="synthesizer",
        system="Senior portfolio strategist. Synthesise research into an investment report.",
        output_schema=InvestmentReport,
    )

    # ── Full pipeline: generalist → parallel sectors → synthesizer ──
    pipeline_sess = LazySession(tracking="basic", console=True)
    full_pipeline = pipeline_sess.as_tool(
        "fan_out_pipeline",
        "Generalist broad sweep → parallel sector specialists → synthesis.",
        mode="chain",
        participants=[generalist, sector_intel_tool, synthesizer],
    )

    result = full_pipeline.run({"task": TASK})
    print(result)

from pydantic import BaseModel, Field
from lazybridge import LazyAgent, LazySession
from lazybridge.core.types import NativeTool

TASK = ("Conduct a full investment analysis for Q2 2025. "
        "Focus on AI infrastructure, energy transition, and CRISPR therapeutics.")

class MarketBriefing(BaseModel):
    sectors: list[str] = Field(description="Sectors analysed")
    summaries: dict[str, str] = Field(description="Key findings per sector")
    overall_outlook: str = Field(description="One-sentence macro outlook")

class RiskProfile(BaseModel):
    overall_sentiment: str = Field(description="bullish | neutral | bearish")
    recommended_action: str = Field(description="invest | hold | avoid | monitor")
    rationale: str

class InvestmentReport(BaseModel):
    title: str
    executive_summary: str
    top_opportunities: list[str] = Field(description="Top 3 opportunities")
    top_risks: list[str] = Field(description="Top 3 risks")
    conclusion: str

def run_variant_4():
    # ── Layer 1: three Google agents in parallel, each → MarketBriefing ──
    market_sess = LazySession(tracking="basic", console=True)
    LazyAgent("google", name="tech_researcher",    session=market_sess,
              native_tools=[NativeTool.WEB_SEARCH],
              output_schema=MarketBriefing,
              system="Technology analyst — AI, semiconductors, cloud. "
                     "Output ONLY valid JSON matching the schema.")
    LazyAgent("google", name="energy_researcher",  session=market_sess,
              native_tools=[NativeTool.WEB_SEARCH],
              output_schema=MarketBriefing,
              system="Energy analyst — renewables, oil & gas, grid. "
                     "Output ONLY valid JSON matching the schema.")
    LazyAgent("google", name="biotech_researcher", session=market_sess,
              native_tools=[NativeTool.WEB_SEARCH],
              output_schema=MarketBriefing,
              system="Biotech analyst — pharma, genomics, medical devices. "
                     "Output ONLY valid JSON matching the schema.")

    parallel_market_tool = market_sess.as_tool(
        "parallel_market_research",
        "Three Google agents research tech, energy, and biotech in parallel.",
        mode="parallel",
    )

    # ── Layers 2 + 3: risk analyst (Anthropic) → report writer (OpenAI) ──
    analysis_sess = LazySession(tracking="basic", console=True)
    risk_analyst  = LazyAgent("anthropic", name="risk_analyst",
                               session=analysis_sess,
                               output_schema=RiskProfile,
                               system="Senior risk analyst. "
                                      "Output ONLY valid JSON matching the schema.")
    report_writer = LazyAgent("openai",    name="report_writer",
                               session=analysis_sess,
                               output_schema=InvestmentReport,
                               system="CFA report writer. "
                                      "Output ONLY valid JSON matching the schema.")

    # ── Full pipeline: parallel research → risk → report — zero def ──
    pipeline = LazySession(tracking="basic", console=True).as_tool(
        "fully_structured_pipeline",
        "Parallel Google research → Anthropic risk assessment → OpenAI report.",
        mode="chain",
        participants=[parallel_market_tool, risk_analyst, report_writer],
    )

    result = pipeline.run({"task": TASK})
    print(result)

Fully Typed Multi-Provider

Every agent owns output_schema. The chain calls json() automatically. Three providers, three Pydantic models, zero manual conversion.

pipeline (chain)
├── parallel_market_tool
│   ├── tech → MarketBriefing
│   ├── energy → MarketBriefing
│   └── biotech → MarketBriefing
├── risk_analyst → RiskProfile
└── report_writer → InvestmentReport

Key pattern

output_schema on the agent makes the chain call json() at each step and pass the serialized Pydantic result to the next participant automatically.
