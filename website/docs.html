Quick Start

Get from zero to a running pipeline in 5 minutes.

Install

bash
pip install lazybridge[anthropic]   # Anthropic / Claude
pip install lazybridge[openai]      # OpenAI / GPT
pip install lazybridge[google]      # Google / Gemini
pip install lazybridge[all]         # all providers

Set your API key

bash
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENAI_API_KEY="sk-..."

export DEEPSEEK_API_KEY="..."

Example 1 — Single call

Python
from lazybridge import LazyAgent

ai = LazyAgent("anthropic")

# Simple text response
answer = ai.text("What is the capital of France?")
print(answer)  # "Paris"

# Full response object (tokens, stop reason, etc.)
resp = ai.chat("Explain quantum entanglement in one sentence.")
print(resp.content)
print(f"Tokens: {resp.usage.input_tokens} in / {resp.usage.output_tokens} out")

Switch provider by changing one string:

Python
ai_openai   = LazyAgent("openai")
ai_google   = LazyAgent("google")
ai_deepseek = LazyAgent("deepseek")

Example 2 — Tool loop

Give the agent a Python function. It will call it automatically when needed.

Python
from lazybridge import LazyAgent, LazyTool

def get_weather(city: str) -> str:
    """Get current weather for a city."""
    return f"Weather in {city}: 22°C, partly cloudy"

ai = LazyAgent("anthropic")
result = ai.loop("What's the weather in Rome and Paris?",
                 tools=[LazyTool.from_function(get_weather)])
print(result.content)

loop() handles the full tool-call cycle: sends message → detects tool call → runs your function → feeds result back → repeats until done.

Example 3 — Multi-agent pipeline

Python
from lazybridge import LazyAgent, LazySession, LazyContext

sess       = LazySession()
researcher = LazyAgent("anthropic", name="researcher", session=sess)
writer     = LazyAgent("openai",    name="writer",     session=sess)

researcher.loop("Find the top 3 developments in AI this week.")

writer.chat(
    "Write a short newsletter section from this research.",
    context=LazyContext.from_agent(researcher),
)

print(sess.events.get())     # full event log
print(sess.store.read_all()) # shared state

Memory & Conversations

Pass a Memory object to keep conversation history automatically. No manual list management.

Python
from lazybridge import LazyAgent, Memory

ai  = LazyAgent("anthropic")
mem = Memory()

ai.chat("My name is Marco", memory=mem)
resp = ai.chat("What's my name?", memory=mem)
print(resp.content)   # "Marco"

print(len(mem))        # 4 — 2 user + 2 assistant messages
print(mem.history)     # list of {"role": ..., "content": ...} dicts
mem.clear()            # reset

The same Memory instance can be shared across multiple agents:

Python
mem = Memory()
agent_a.chat("Remember: the project deadline is Friday", memory=mem)
agent_b.chat("What's the deadline?", memory=mem)   # answers "Friday"

Persistence across restarts

Python
import json
from lazybridge import LazyAgent, Memory, LazySession

sess = LazySession(db="chat.db")
ai   = LazyAgent("anthropic", session=sess)

# Restore previous session
raw = sess.store.read("history")
mem = Memory.from_history(json.loads(raw)) if raw else Memory()

ai.chat("continue from where we left off", memory=mem)

# Save at the end
sess.store.write("history", json.dumps(mem.history))

Tools

Tools let the LLM call your code. LazyTool handles schema generation, execution, and validation automatically.

Wrapping a Python function

Python
from lazybridge import LazyTool

def get_weather(city: str, unit: str = "celsius") -> str:
    """Get current weather for a city."""
    return f"Weather in {city}: 22°C, sunny"

tool = LazyTool.from_function(get_weather)
# Name from function name, description from docstring, schema from type hints

Type hint → JSON Schema

Python annotationJSON Schema
str{"type":"string"}
int{"type":"integer"}
float{"type":"number"}
bool{"type":"boolean"}
list[X]{"type":"array","items":<X>}
dict{"type":"object"}
Optional[X] / X | None{"anyOf":[<X>,{"type":"null"}]}
Literal["a","b"]{"enum":["a","b"]}
MyModel (Pydantic)Full JSON Schema
Annotated[X, "desc"]{<X>, "description":"desc"}

Parameters are required when no default; optional when param=default.

Adding descriptions with Annotated

Python
from typing import Annotated

def search_web(
    query: Annotated[str, "The search query. Be specific, use keywords."],
    language: Annotated[str, "ISO 639-1 language code, e.g. 'en', 'it'"] = "en",
    max_results: Annotated[int, "Maximum results (1–50)"] = 10,
) -> list[str]:
    ...

tool = LazyTool.from_function(search_web)

Schema modes

SIGNATURE (default): reads type hints. HYBRID: types from code, descriptions from LLM. LLM: full schema inferred by LLM (for legacy code without annotations).

Python
from lazybridge import LazyTool, ToolSchemaMode, LazyAgent

llm  = LazyAgent("anthropic")
tool = LazyTool.from_function(
    search_web,
    schema_mode=ToolSchemaMode.HYBRID,
    schema_llm=llm,
).compile()   # LLM call happens here, not at first loop step

Guidance — hints for the calling model

Python
tool = LazyTool.from_function(
    get_weather,
    guidance="Call this before answering any weather question. Always ask for the city if not provided.",
)

Sessions & Pipelines

A LazySession is the shared container for a multi-agent pipeline. Create one when you have more than one agent that needs to share state, tracking, or a graph.

ScenarioSession needed?
Single agent, one-off questionNo
Two agents sharing resultsYes
Agents running concurrentlyYes
Persistent state across runsYes
GUI pipeline visualizationYes

Creating a session

Python
from lazybridge import LazySession

sess = LazySession()                        # in-memory
sess = LazySession(db="my_pipeline.db")    # persistent SQLite
sess = LazySession(tracking="verbose")     # all events including stream chunks

Event tracking — embedded, zero infra

Every LazySession automatically records every model request, tool call, loop step, and response in a built-in SQLite event log. No cloud account. No OTel collector. No configuration beyond the tracking= level.

LevelEvents recorded
OFFNothing logged
BASIC (default)model_request, model_response, tool_call, tool_result, tool_error, agent_start, agent_finish, loop_step
VERBOSE / FULLAll of the above + messages, system_context, stream_chunk
Python
from lazybridge import LazySession, Event

sess = LazySession(db="pipeline.db", tracking="verbose", console=True)

# ... run your pipeline ...

# Query from Python — SQLite indexes on agent_id + event_type
all_events    = sess.events.get()
tool_calls    = sess.events.get(event_type=Event.TOOL_CALL)
writer_events = sess.events.get(agent_id=writer.id)
recent_50     = sess.events.get(limit=50)

for ev in tool_calls:
    print(ev["timestamp"], ev["agent_name"], ev["event_type"], ev["data_json"])

vs LangSmith: cloud account + per-event pricing — LazyBridge uses a local SQLite file, always free.
vs OTel: requires a collector and exporter infrastructure — LazyBridge has zero external dependencies, works offline.
vs LangChain callbacks: manual wiring per chain — LazyBridge tracking is automatic on every LazySession.

Concurrent agents with gather()

Python
import asyncio
from lazybridge import LazyAgent, LazySession

sess    = LazySession()
agent_a = LazyAgent("anthropic", name="news_a", session=sess)
agent_b = LazyAgent("openai",    name="news_b", session=sess)
agent_c = LazyAgent("google",    name="news_c", session=sess)

async def run():
    results = await sess.gather(
        agent_a.aloop("Summarise AI news from the US"),
        agent_b.aloop("Summarise AI news from Europe"),
        agent_c.aloop("Summarise AI news from Asia"),
    )
    for r in results:
        print(r.content[:200])

asyncio.run(run())

Context Injection

LazyContext lets you inject dynamic content into an agent's system prompt — evaluated at call time, not at construction time.

Four source types

Python
from lazybridge import LazyContext, LazyStore
from datetime import date

# 1. Static text
ctx = LazyContext.from_text("Always answer in formal English.")

# 2. Function (called at execution time)
ctx = LazyContext.from_function(lambda: f"Today is {date.today().isoformat()}")

# 3. From shared store
store = LazyStore()
ctx = LazyContext.from_store(store, keys=["research", "style_guide"])

# 4. From another agent's last output
researcher = LazyAgent("anthropic", name="researcher")
ctx = LazyContext.from_agent(researcher)
# Reads researcher._last_output when invoked; "" if agent not yet run

Composing contexts

Python
ctx = (
    LazyContext.from_text("You are a senior analyst.")
    + LazyContext.from_agent(data_collector)
    + LazyContext.from_store(sess.store, keys=["market_data"])
    + LazyContext.from_function(lambda: f"Today: {date.today()}")
)
# Sources evaluated in order, joined with blank lines

Agent-level vs call-level

Python
# Context applied to every call on this agent
writer = LazyAgent("openai", context=LazyContext.from_text("Write in Italian."))
writer.chat("What is Python?")   # "Python è un linguaggio..."

# Override for a single call
writer.chat("What is Python?", context=LazyContext.from_text("Write in German."))

Full pipeline example

Python
from lazybridge import LazyAgent, LazyContext, LazyStore

store = LazyStore()

# Contexts declared once — evaluated lazily when each agent runs
collector = LazyAgent("anthropic", name="collector")
analyst   = LazyAgent("anthropic", name="analyst",
                      context=LazyContext.from_store(store, keys=["papers"]))
writer    = LazyAgent("openai",
                      context=(
                          LazyContext.from_text("Write for a non-technical audience.")
                          + LazyContext.from_store(store, keys=["findings"])
                      ))

# Pure execution + store handoffs
collector.loop("Collect the top 5 AI papers this month")
store.write("papers", collector._last_output)

analyst.chat("Identify the 3 most impactful findings")
store.write("findings", analyst._last_output)

writer.chat("Write a blog post from these findings")

Testing your context

Python
# ctx() returns a plain string — testable independently
print(ctx())              # before agents run: only static parts
researcher.loop("find data")
print(ctx())              # now includes researcher's output

Routing

LazyRouter routes the result of one agent to one of several agents based on a condition. For simple branching, a plain Python if/else is fine — use LazyRouter when you want the branching to appear in the graph schema or have 3+ named routes.

Basic usage

Python
from lazybridge import LazyAgent, LazyRouter

writer   = LazyAgent("anthropic", name="writer")
reviewer = LazyAgent("openai",    name="reviewer")

router = LazyRouter(
    condition=lambda response: "writer" if "APPROVED" in response.upper() else "reviewer",
    routes={"writer": writer, "reviewer": reviewer},
    name="approval_gate",
)

checker = LazyAgent("anthropic", name="checker")
result  = checker.chat("Evaluate this draft: [...]")
next_agent = router.route(result.content)
final = next_agent.chat("Proceed with: " + result.content)

Multiple routes with fallback

Python
def pick_route(response: str) -> str:
    r = response.strip().lower()
    if "research" in r:   return "research"
    if "analyse"  in r:   return "analyse"
    if "write"    in r:   return "write"
    return "review"

router = LazyRouter(
    condition=pick_route,
    routes={"research": researcher, "analyse": analyst,
            "write": writer, "review": reviewer},
    name="task_router",
    default="review",  # fallback if condition returns unknown key
)

Async condition (LLM-based routing)

Python
import asyncio
from lazybridge import LazyAgent, LazyRouter

classifier = LazyAgent("anthropic")  # created once, reused

async def classify_with_llm(text: str) -> str:
    label = await classifier.atext(
        f"Classify as: research / analyse / write. Task: {text}. Return only the label."
    )
    return label.strip().lower()

router = LazyRouter(
    condition=classify_with_llm,
    routes={"research": researcher, "analyse": analyst, "write": writer},
    default="write",
)

next_agent = asyncio.run(router.aroute("What are the latest GPU benchmark numbers?"))

Full pipeline with revision loop

Python
from lazybridge import LazyAgent, LazyRouter, LazySession

sess = LazySession()
drafter   = LazyAgent("anthropic", name="drafter",   session=sess)
reviewer  = LazyAgent("openai",    name="reviewer",  session=sess)
publisher = LazyAgent("anthropic", name="publisher", session=sess)

router = LazyRouter(
    condition=lambda r: "publisher" if "APPROVED" in r.upper() else "reviewer",
    routes={"publisher": publisher, "reviewer": reviewer},
    default="reviewer",
)

content = "Write a blog post about AI safety."
for _ in range(3):   # up to 3 revision cycles
    draft = drafter.chat(content)
    check = reviewer.chat(f"Review and say APPROVED or REJECTED with reason: {draft.content}")
    next_agent = router.route(check.content)
    if next_agent is publisher:
        result = publisher.chat(f"Publish: {draft.content}")
        print("Published:", result.content[:200])
        break
    else:
        content = f"Revise based on: {check.content}\n\nOriginal: {draft.content}"
